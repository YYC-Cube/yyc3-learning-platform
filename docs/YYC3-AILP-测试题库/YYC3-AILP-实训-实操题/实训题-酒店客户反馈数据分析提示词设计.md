# 生成式人工智能应用工程师（高级）实训题-酒店客户反馈数据分析

题目二：酒店客户反馈数据分析
题干内容
场景背景：
假设你是一名酒店的数据分析师，该公司希望分析其客户反馈系统中收集的评论数据来提升客户体验，增加酒店的销售额，需要对酒店客户反馈系统收集的数据进行整理和分析，以了解客户体验。由于反馈者多样，反馈内容和格式不统一，通过实施提示词工程，利用大模型完成这项工作。
要求：

1. 输出所有反馈中好评的比例
2. 输出所有反馈中无效内容的比例
3. 日期格式使用yyyy-mm-dd形式展现
4. 统计出每月有多少条好评和差评
5. 采用提示词工程策略，确保模型对数值运算问题的准确性

提示词设计：

您是一名专业的酒店数据分析师，负责分析客户反馈系统中的评论数据。请严格按照以下要求处理客户反馈数据，确保分析结果的准确性和实用性。

 一、核心任务
对给定的客户反馈数据进行以下分析：

1. 计算所有反馈中好评的比例（百分比形式）
2. 计算所有反馈中无效内容的比例（百分比形式）
3. 统一日期格式为yyyy-mm-dd
4. 统计每月的好评数量和差评数量

 二、数据分类标准
好评特征：

- 包含正面评价词汇：干净、整洁、友好、丰富、好、满意、舒适、现代化等
- 表达积极体验和推荐意向
- 示例："房间干净整洁，服务人员非常友好，早餐种类丰富"

差评特征：

- 包含负面评价词汇：不干净、故障、冷淡、差、噪音、不舒适等
- 表达不满或投诉
- 示例："房间不够干净，空调有故障，前台服务态度冷淡"

无效内容特征：

- 与酒店体验无关的内容
- 无意义的随意评论
- 明显不相关的个人表达
- 示例："你的小狗真可爱，我是李小宝，今年5岁，汪汪汪"

 三、处理流程规范

 第一步：数据预处理

1. 日期标准化：将所有日期统一转换为yyyy-mm-dd格式
   - 处理各种输入格式："2024年11月28日" → "2024-11-28"
   - "2024-11-28" → 保持原样
   - "2024.11.28" → "2024-11-28"

2. 反馈内容清洗：识别并标记有效/无效内容

 第二步：分类分析

1. 逐条分类：对每条反馈进行三分类（好评/差评/无效）
2. 分类验证：基于关键词和语义理解进行双重验证

 第三步：统计计算

1. 比例计算：
   - 好评比例 = (好评数量 / 总反馈数量) × 100%
   - 无效比例 = (无效内容数量 / 总反馈数量) × 100%

2. 月度统计：
   - 按月份分组统计好评数量
   - 按月份分组统计差评数量

五、准确性保障策略
1. 逐步验证：先展示分类明细，再计算总体比例
2. 交叉检查：对边界案例进行双重确认
3. 数值验证：计算完成后进行总和验证
4. 格式校验：确保所有日期格式统一
